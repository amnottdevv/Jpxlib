# Contributing

Thank you for your interest in contributing to **JPX**.

Contributions help improve the language, ecosystem, and documentation.

---

# Ways to Contribute

You can contribute in several ways:

- Modifying the core code
- Improving documentation
- Reporting bugs
- Suggesting new features
- Creating new modules

---

# Modifying the Code

If you want to modify the JPX source code:

1. Fork the repository
2. Create a new branch
3. Implement your changes
4. Test your changes
5. Submit a Pull Request

Example workflow:

```
git fork https://github.com/alzzmetth/jpx-lang
git checkout -b feature-update
git commit -m "Improve syntax handling"
git push origin feature-update
```

---

# Adding Modules

JPX supports expanding functionality through modules.

If you want to add a module:

1. Create a new module file
2. Follow the existing module structure
3. Ensure the module works correctly
4. Document the module usage

Example module concept:

```
Module example;

Function hello() {
    Print "Hello from module";
}
```

---

# Reporting Issues

If you discover a bug or unexpected behavior:

1. Open an Issue in the repository
2. Describe the problem clearly
3. Include example code if possible
4. Mention the JPX version used

---

# Code Style

Please follow these principles:

- Keep syntax clear and readable
- Maintain consistent formatting
- Avoid unnecessary complexity
- Document important parts of the code
